Overview

Tango infrastructure and services are divided into objects that allow for fast human identification of service infrastructure and provide a basis for automated instrumentation. The object identifiers are codified according to the convention below using ASCII latin characters and digits. The object identifier is stored in one of two places - in legacy world we store this information in the hostname of a server instance. In the new AWS world we store this information in the key-value metadata (called "Tags") that is added to corresponding objects within an AWS region. 

Domains

Internal production and staging: tangome.gbl

Internal dev, office, other non-production environments: tango.corp

External all (except Swift): tango.me

External Swift: tangome.net

Common Objects

Region - represents a geographical location throughout the world and is normally used to identify countries where Tango services reside

Site - represents a physical datacenter location within a region that hosts one or more Tango environments 

Environment - represents an independent logical location that provides one or more Tango services whose dependencies are contained within the environment

Cluster - represents a logical grouping of services within an environment, this identifier is an arbitrary construct that can be used to group differences within the service stack that would otherwise be identical

Service/Service Type - represents a unique service or an application within Tango infrastructure

Host Number - sequence number of the host, this identifier is meant to provide uniqueness to each server or service instance providing the same service type within legacy hostname framework

Conventions

Example: 

us0101afc001 is the first facilitator server, of cluster a, in the first environment of the first datacenter in the United States


Hostname:

us - Region

us01 - Site

us0101 - Environment

us0101a - Cluster

us0101afc - Service Type

us0101afc001 - Host Number

Clusters

Cluster represents a logical grouping of services within an environment, this identifier is an arbitrary construct that can be used to group differences within the service stack that would otherwise be identical. Clusters are numbered a-z, where each letter represents cluster within an environment. Databases use cluster designators to identify either failover precedence or shard generation, depending on the database. Remaining service clusters follow specific cluster designations below.

Unless otherwise noted, letter 'a' is used by default for all other clusters

Host Number

In non-AWS systems, the numeric designators 000-999 represent the number of the host carrying the same function within same cluster, within same datacenter. The two Connection Manger servers running side-by-side may have 001 and 002 Host number designations. The host designator within legacy AWS includes a full IP address of the host to ensure uniqueness and consists of 12 digits 111222333444. The host designator within new AWS is not used as uniqueness is ensured by AWS provided in this case.

AWS-specific Objects

There are additional object definitions that exist in Amazon AWS service infrastructure that are not otherwise present in other infrastructure systems. All AWS objects are required to have specific set of Tags that describe the region, site, environment, cluster and service that the AWS object is associated with. All objects in AWS have the same required set of Tags. Example values below are for adInfo service in us0601 environment:

Key
Value
regionus
site06
environment01
clustera
serviceao

* service tag can contain multiple service identifiers, comma delimited

Instances

Instances are virtual servers within AWS infrastructure. Each instance must be tagged, following the the set of Tags and naming convention above. Name Tags and hostnames are also significant to instance objects in AWS, in order to be compatible with existing Tango Puppet systems. In addition to Tags, instance hostnames need to follow the naming convention. The Host Number for instances is replaced by string representing a zero-padded IP address to ensure unique constraint - 172016012110. Example name for adInfo instance in us0601 environment below:

us0601aao172016012110

Amazon Machine Images

Amazon Machine Images (AMIs for short) are full-instance snapshots, capturing the state of the system at the time snapshot was taken. AMIs serve as the source for launching new instances within Auto Scaling Groups, as defined by Launch Configurations. Each AMI must be tagged according to the application that it's servicing, including the set of Tags with values from naming convention above. Additional Tags identifying version of the software versions contained are required. Each additional Tag contains the service identifier in the Key and the version of the service deployed to the AMI. In addition to Tags, AMI names need to follow the naming convention. The Host Number for AMIs is replaced by timestamp string in the format YYYYmmddHHMMSSffffff (Year, Month, Day, Hour, Minute, Second, Microsecond). Example additional Tags and name for adInfo AMI in us0601 environment below:

Elastic Load Balancers

Elastic Load Balancers (ELBs for short) are load-balancer objects within AWS infrastructure. Each ELB must be tagged according to the application that it's servicing (and it's location, etc..), including the set of Tags with values from naming convention above. In addition to Tags, ELB names need to follow the naming convention. The Host Number for ELBs is replaced by string "-vip" and a sequence number to ensure unique constraint - vip000-999. Example name for adInfo ELB in us0601 environment below:

us0601aao-vip001

Launch Configurations

Launch Configurations (LCs for short) are policy objects, defining behavior of Auto Scaling Groups. Each LC must be named according to the application that it's servicing (and it's location, etc..), using naming convention above. LC objects do not support tags. The LC names need to follow the naming convention. The Host Number for LCs is replaced by timestamp string in the format YYYYmmddHHMMSSffffff (Year, Month, Day, Hour, Minute, Second, Microsecond). Example name for adInfo LC in us0601 environment below:

us0601aao-2015010123590101012345

Auto Scaling Groups

Auto Scaling Groups (ASGs for short) are grouping objects that define application cluster scaling behavior for the underlying instances, based on supplied Launch Configuration and defined event thresholds. Each ASG must be tagged according to the application that it's servicing (and it's location, etc..), including the set of Tags with values from naming convention above. In addition to Tags, ASG names need to follow the naming convention. The Host Number for ASGs is replaced by timestamp string in the format YYYYmmddHHMMSSffffff (Year, Month, Day, Hour, Minute, Second, Microsecond). Example name for adInfo ASG in us0601 environment below:

us0601aao-2015010123590101012345

Security groups

Security groups (SGs for short) are objects that specify L4 port access policies for other object within AWS. Each SG must be tagged according to the application that it is servicing, including the set of Tags with values from naming convention above. In addition to Tags, the security groups must be named according to the naming convention above. The Host Number for SGs is replaced by timestamp string in the format YYYYmmddHHMMSSffffff (Year, Month, Day, Hour, Minute, Second, Microsecond). Example name for adInfo SG in us0601 environment below:

us0601aao-2015010123590101012345

RDS Clusters

Relational Database Service clusters (RDS for short) are MySQL database service equivalents in Amazon. Each RDS exposes a single cluster name and endpoint, regardless of how many actual server instances are in the cluster. In addition to standard set of Tags, each RDS cluster must be named according to naming convention above. Example RDS Cluster name for nudityDetector database is below:

us0701ansd001

Network-specific Objects

  • Core router
  • Standby router
  • Core switch
  • Standby switch
  • Primary Load Balancer
  • Backup Load Balancer
  • Top of rack switches
